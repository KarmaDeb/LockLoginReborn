Creating my first event

The only thing you will need is basic knowledge of java, nothing more.

Step One

Create a class with the name of the event, which extends eu.locklogin.api.module.plugin.api.event.util.Event

public final class ClientFastAuthEvent extends Event {
     
     private final Object eventObj;
     private boolean handled = false;
     
     /**
     * Initialize the event
     *
     * If your event is called from another
     * event, please make use of the "getEvent()"
     * method, so external APIs can use it to reflect
     * into the original event. Otherwise, parse eventOwner
     * as null.
     */
     public ClientFastAuthEvent(final Object eventOwner) {
         eventObj = eventOwner;
     }
     
     /**
     * Check if the event has been handled
     *
     * @return if the event has been handled
     */
     @Override
     public final boolean isHandled() {
         return handled;
     }

     /**
     * Set the event handle status
     *
     * @param status the handle status
     */
     @Override
     public final void setHandled(boolean status) {
         handled = status;
     }
     
     /**
     * Get the event instance
     *
     * @return the event instance
     */
    @Override
    public final @Nullable Object getEvent() {
        return eventObj;
    }
}

Registering an event listener

You can also listen to other LockLogin events or module events by creating an EventListener

public class JoinListener implements EventListener {

    private final static Map<String, BPremiumUtils> utilMap = new HashMap<>();

    @ModuleEventHandler(priority = ModuleEventHandler.Priority.LAST)
    public void onJoin(UserPreJoinEvent e){
        ModulePlayer player = e.getPlayer();
        MyConfigurationObject configuration = new MyConfigurationObject();
        if (configuration.hasFastLogin(player.getUUID())) {
            ClientFastAuthEvent event = new ClientFastAuthEvent(e, player);
            ModulePlugin.callEvent(event);

            if (!event.isHandled()) {
               player.requestLogin();
            }
        }
    }
}

After creating the EventListener

public class Main extends PluginModule {

    /**
     * On module enable logic
     */
    @Override
    public void enable() {
        getPlugin().registerListener(new JoinListener());
    }

    /**
     * On module disable logic
     */
    @Override
    public void disable() {
        getPlugin().unregisterListener(new JoinListener());
    }
}

Handling events

When using the API, you may have noticed there're 3 extra methods, in all the event objects, which are

Event.isHandleable() //Returns if the event can be handled
Event.isHandled() //Returns if the event is handled
Event.getHandleReason() // Returns the event handle reason
Event.setHandled(Boolean, String) // Set the event handle status and reason
Event.getEvent() // Get the event parent event, this can be another LockLogin event, bukkit event, bungee event, or even velocity event

What is handling an event in LockLogin

Handling an event in LockLogin means you have taken the event and you've done some action on it or some of its recipients.

For example, UserJoinEvent. By handling this event, the client would be kicked with the handle reason. Why? Handling an event is like stopping it to keep going, and stopping joining an event means stopping client connection requests.
